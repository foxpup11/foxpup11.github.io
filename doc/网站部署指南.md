## 项目结构

升级后的网站采用**纯静态HTML/CSS/JS**方案，无需任何构建工具，直接部署到GitHub Pages即可。

```
foxpup11.github.io/
├── index.html          # 首页（英雄区 + 技能 + 项目概览 + 经历 + 联系方式）
├── about.html          # 关于我（教育 + 经历 + 技能详情）
├── projects.html       # 项目详情页（完整项目列表 + 分类筛选）
├── css/
│   └── style.css       # 主样式文件（CSS变量 + 组件样式 + 响应式）
├── js/
│   └── script.js       # 交互脚本（导航 + 动画 + 筛选）
├── doc/
│   └── 网站部署指南.md  # 本文档
└── README.md           # 项目说明
```

## 本地预览

### 方法一：VSCode Live Server（推荐）

1. 在VSCode中安装 **Live Server** 扩展
2. 右键点击 `index.html` → **Open with Live Server**
3. 浏览器自动打开 `http://127.0.0.1:5500`

### 方法二：Python HTTP Server

```bash
cd foxpup11.github.io
python -m http.server 8000
# 浏览器打开 http://localhost:8000
```

### 方法三：直接双击打开

直接双击 `index.html` 文件即可在浏览器中查看（部分动画效果可能受限）。

## 部署到GitHub Pages

### 步骤1：提交代码

```bash
cd foxpup11.github.io

# 添加所有文件
git add .

# 提交
git commit -m "feat: upgrade to enterprise-level personal website"

# 推送到GitHub
git push origin main
```

### 步骤2：配置GitHub Pages

1. 打开你的仓库：`https://github.com/foxpup11/foxpup11.github.io`
2. 点击 **Settings** → **Pages**
3. 在 **Source** 部分：
   - 选择 **Deploy from a branch**
   - Branch 选择 `main`，目录选择 `/ (root)`
   - 点击 **Save**

### 步骤3：访问网站

等待1-2分钟，访问 `https://foxpup11.github.io` 即可看到升级后的网站。

### 自定义域名（可选）

如果你有自己的域名：

1. 在仓库根目录创建 `CNAME` 文件，内容为你的域名
2. 在域名服务商处添加DNS记录：
   ```
   类型: CNAME
   名称: @ 或 www
   值: foxpup11.github.io
   ```
3. 在GitHub仓库 Settings → Pages 中填写自定义域名

## 常见问题

### Q: 网站样式没有加载？

**A:** 检查文件路径是否正确。确保 `css/style.css` 和 `js/script.js` 文件存在于正确位置。GitHub Pages 有几分钟的缓存延迟，稍等片刻刷新即可。

### Q: 中文字体显示不正常？

**A:** 网站使用了 Google Fonts 的 Inter 和 Noto Sans SC 字体。如果在国内访问较慢，可以在 `index.html` 的 `<head>` 中移除 Google Fonts 链接，使用系统默认字体。

### Q: 如何添加照片作为头像？

**A:** 
1. 将照片放入 `images/` 目录（需要新建）
2. 修改 `index.html` 中的 `.hero-avatar` 部分：

```html
<div class="hero-avatar">
  <img src="images/avatar.jpg" alt="司震" style="width:100%;height:100%;object-fit:cover;border-radius:50%;">
</div>
```

3. 在 `css/style.css` 中调整 `.hero-avatar` 的样式

### Q: 如何添加新的页面？

**A:** 
1. 复制现有的 `about.html` 作为模板
2. 修改页面内容
3. 在导航栏的 `nav-links` 中添加新链接
4. 在页脚的 `footer-links` 中添加链接

### Q: 移动端显示异常？

**A:** 网站已适配移动端响应式布局。如果仍有问题，检查 `css/style.css` 底部的 `@media` 查询部分。
